── LOOP ──
Read state → battleship → prestige → plan → overnight.
Pick next unclosed gap. Verify claim against C source first.
Implement. Build. Suite. Doc sweep ALL walkways version-bumped.
Barnacle hunt stale numbers across every .md + README + BANNER.

── DOC DISCIPLINE ──
After EVERY implementation: update ALL walkway files (state,
goal-mantra, prestige, overnight, plan, battleship) AND README
AND BANNER AND entry.md AND index.md AND testing.md AND vault.
Version-bump all in lockstep. Grep for old numbers across the
entire markdown tree after every batch.

── VAULT RULES ──
Resolved gaps → vault/achievements.md with file:line evidence.
Stale claims discovered during verification → vault with explanation.
Battleship carries ONLY active gaps — no checkmarks, no stale items.
Vault is history. Battleship is TODO. Keep them cleanly separated.

── MEMORY RULES ──
Save to memory: user preferences, env facts, tool quirks, stable
conventions, project-level patterns. Durable facts across sessions.
Do NOT save to memory: task progress, PR numbers, commit SHAs,
session outcomes, TODOs, version numbers, gap counts. These live
in vault (historical record) and walkway files (state).
session_search for cross-session recall. Skills for procedural patterns.
Memory at capacity → consolidate, don't expand.

── NO TIME ──
No dates. No pacing. No duration estimates. No human-week framing.
AI is time-irrelevant. This is agent work. Work until done.
The next session picks up where this one left off.

── BOUNDARIES ──
Skills hold REUSABLE patterns only. No project-specific paths,
file trees, version numbers, commit counts, or milestone percents.
Those belong in memory or mind-palace. Skills must survive a
different project starting fresh.

── KEEP WORKING ──
Pick the next unclosed gap. Verify claim against C source before
implementing — many claims are stale. Force one edit, build,
commit. The loop breaks on the first compilation.
Report result, don't ask permission. Never stop.
Awaiting direction only when truly exhausted — not between gaps.
